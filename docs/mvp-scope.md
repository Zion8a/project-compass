# MVP Scope – Project Compass

## Purpose of the MVP

The purpose of the first version is to prove the core idea of Project Compass:

A project management tool should help users understand and structure a project before they start managing tasks.

The MVP should be small, clear and usable.

It should not try to compete with Trello, Jira, Asana or ClickUp in all areas. It should focus on the project start and the connection between purpose, risks, decisions and work.

## MVP goal

The first version should allow a user to:

1. Create a new project
2. Answer a short project interview
3. Generate a project map
4. Add tasks
5. Add risks
6. Add decisions
7. View a simple project board
8. Generate a simple status report

## Included in version 0.1

### Project interview

The user should answer questions about:

- project name
- purpose
- goal
- deliverables
- deadline
- stakeholders
- risks
- important decisions
- success criteria

### Project map

The system should display:

- project name
- purpose
- goals
- deliverables
- top risks
- important decisions
- next steps

### Task board

The first board should have these columns:

- Backlog
- Planned
- In progress
- Blocked
- Review
- Done

### Risks

The user should be able to create simple risks with:

- title
- description
- probability
- impact
- action
- owner
- status

### Decisions

The user should be able to create simple decisions with:

- title
- description
- status
- owner
- deadline
- consequence

### Status report

The system should generate a simple report with:

- current status
- completed work
- blocked work
- top risks
- decisions needed
- next steps

## Not included in version 0.1

The first version will not include:

- user accounts
- database
- team collaboration
- comments
- notifications
- file uploads
- budget management
- Gantt charts
- integrations with other systems
- advanced AI features
- mobile app
- role-based permissions

## Storage in version 0.1

Version 0.1 will store data locally in the browser.

This means:

- no login is required
- no server is required
- data is only available on the same device and browser
- the goal is to test the concept, not to build a full production system

## Definition of done for MVP

The MVP is done when a user can:

- open the app
- create a project
- answer the project interview
- see a generated project map
- create at least one task
- create at least one risk
- create at least one decision
- view the project board
- generate a basic status report

## Success criteria

The MVP is successful if a first-time user understands:

- what the project is about
- why the project exists
- what needs to be done
- what the main risks are
- what decisions are needed
- what the next step is
