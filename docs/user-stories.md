# User Stories – Project Compass

## Epic 1: Project start

### US-001 – Create a new project

As a user,  
I want to create a new project,  
so that I can start structuring my work.

Acceptance criteria:

- The user can enter a project name.
- The user can start a project interview.
- The project is created when the interview is completed.
- The project is visible on the project map.

Priority: High

---

### US-002 – Answer project interview

As a user,  
I want to answer a short set of project questions,  
so that the system can help me create structure before I start working.

Acceptance criteria:

- The user can answer questions about purpose, goal, deliverables, deadline, stakeholders, risks, decisions and success criteria.
- Required fields are clearly marked.
- The user can submit the interview.
- The answers are used to generate the project map.

Priority: High

---

## Epic 2: Project map

### US-003 – View project map

As a user,  
I want to see a project map,  
so that I can understand the project’s purpose, goals, risks, decisions and next steps.

Acceptance criteria:

- The project map shows project name.
- The project map shows purpose.
- The project map shows goals.
- The project map shows deliverables.
- The project map shows top risks.
- The project map shows important decisions.
- The project map shows next steps.

Priority: High

---

## Epic 3: Tasks

### US-004 – Create task

As a user,  
I want to create a task,  
so that I can describe work that needs to be done.

Acceptance criteria:

- The user can create a task with title and description.
- The user can select a status.
- The task appears on the board.
- The task can be moved between board columns.

Priority: High

---

### US-005 – View task board

As a user,  
I want to see tasks on a board,  
so that I can understand what is planned, ongoing, blocked and done.

Acceptance criteria:

- The board has the columns Backlog, Planned, In progress, Blocked, Review and Done.
- Tasks are displayed in the correct column.
- The board is easy to scan visually.

Priority: High

---

## Epic 4: Risks

### US-006 – Create risk

As a user,  
I want to create a risk,  
so that I can identify what might go wrong in the project.

Acceptance criteria:

- The user can create a risk with title and description.
- The user can set probability.
- The user can set impact.
- The user can describe an action.
- The risk appears in the risk view.

Priority: High

---

## Epic 5: Decisions

### US-007 – Create decision

As a user,  
I want to create a decision,  
so that important choices do not disappear in meetings, comments or chat messages.

Acceptance criteria:

- The user can create a decision with title and description.
- The user can set status.
- The user can set owner.
- The user can set deadline.
- The decision appears in the decision view.

Priority: High

---

## Epic 6: Status report

### US-008 – Generate status report

As a user,  
I want to generate a simple status report,  
so that I can communicate the project status to others.

Acceptance criteria:

- The report includes current project status.
- The report includes completed work.
- The report includes blocked work.
- The report includes top risks.
- The report includes decisions needed.
- The report includes next steps.

Priority: Medium

---

## Epic 7: Local storage

### US-009 – Save project locally

As a user,  
I want my project data to be saved in the browser,  
so that I do not lose my work when I reload the page.

Acceptance criteria:

- Project data is saved in local storage.
- Project data is still available after page reload.
- No login is required in version 0.1.

Priority: High
